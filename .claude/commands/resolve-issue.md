# GitHub 이슈 해결 계획

## 이슈 번호
$ARGUMENTS

## 지침

위 이슈 번호를 바탕으로 해결 계획을 수립해주세요.

### 1단계: 이슈 가져오기
`gh issue view $ARGUMENTS` 명령어로 이슈 내용을 확인합니다.

### 2단계: 이슈 분석
- 이슈의 목적과 요구사항 파악
- 관련 코드베이스 탐색 (파일, 컴포넌트, 함수 등)
- 영향 범위 분석

### 3단계: 해결 계획 수립
다음 형식으로 상세 계획을 작성:

```markdown
## 이슈 요약
- **제목**: [이슈 제목]
- **목적**: [해결하려는 핵심 문제]

## 현재 상태 분석
- 관련 파일: [파일 목록]
- 현재 동작: [현재 어떻게 동작하는지]
- 문제점/개선점: [무엇이 부족한지]

## 구현 계획

### Task 1: [작업명]
- **목표**: ...
- **변경 파일**: ...
- **상세 내용**:
  - [ ] 세부 작업 1
  - [ ] 세부 작업 2

### Task 2: [작업명]
- **목표**: ...
- **변경 파일**: ...
- **상세 내용**:
  - [ ] 세부 작업 1
  - [ ] 세부 작업 2

## 실행 순서
1. Task 1 (독립)
2. Task 2 (Task 1 완료 후)
...

## 검증 방법
- [ ] 테스트 항목 1
- [ ] 테스트 항목 2

## 리스크 및 고려사항
- ...
```

### 4단계: 사용자 검토 요청
계획을 제시한 후 반드시 사용자에게 다음을 확인:
1. 계획이 이슈의 요구사항을 충족하는지
2. 구현 방향에 동의하는지
3. 추가하거나 수정할 부분이 있는지

**사용자 승인 전까지 코드 작성을 시작하지 마세요.**

### 5단계: 승인 후 작업
사용자가 계획을 승인하면:
- TodoWrite 도구로 작업 목록 생성
- 순서대로 작업 진행
- 완료 후 이슈 닫기 여부 확인
