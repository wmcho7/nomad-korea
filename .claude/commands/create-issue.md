# GitHub 이슈 생성

## 요청 내용
$ARGUMENTS

## 지침

위 요청을 바탕으로 GitHub 이슈를 생성해주세요.

### 1단계: 요청 분석
- 요청 내용을 분석하여 이슈의 성격 파악 (feature, bug, enhancement, docs 등)
- 관련 코드나 파일이 있다면 코드베이스 탐색

### 2단계: 이슈 내용 작성
다음 형식으로 이슈를 구성:

**제목**: 간결하고 명확하게 (50자 이내 권장)
- feature: `feat: [기능명]`
- bug: `fix: [버그 설명]`
- enhancement: `enhance: [개선 내용]`
- docs: `docs: [문서 내용]`

**본문 구조**:
```markdown
## 개요
[이슈의 배경과 목적을 1-2문장으로]

## 상세 내용
[구체적인 요구사항이나 문제 설명]

## 작업 항목
- [ ] 작업 1
- [ ] 작업 2
- [ ] 작업 3

## 참고 사항
[관련 파일, 의존성, 주의사항 등]
```

### 3단계: 이슈 생성
`gh issue create` 명령어를 사용하여 이슈를 생성합니다.

### 주의사항
- 이슈 생성 전 사용자에게 제목과 본문을 보여주고 확인 받기
- 레이블이 필요한 경우 적절한 레이블 추가 (`--label` 옵션)
- 담당자 지정이 필요하면 `--assignee` 옵션 사용
